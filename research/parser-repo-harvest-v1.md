# Parser Repo Harvest V1

Date: 2026-06-06

This note extends the native parser/import roadmap beyond the current lightweight fixture set:
`javascript`, `typescript`, `python`, `rust`, `c`, `cpp`, `java`, `go`, `swift`, `csharp`, `php`, `ruby`, `kotlin`, `scala`, `dart`, `lua`, `shell`, `sql`, `zig`, `elixir`, `erlang`, `haskell`, and `r`.

The goal is not to add parser dependencies to this facade. The immediate use is to map primary parser/compiler AST sources to the Frontier import evidence fields that a host adapter can populate:

- `nativeAst.parser`, `nativeAst.metadata.astFormat`, and `nativeAst.nodes[*].kind/fields/sourceSpan`
- `semanticIndex.documents`, `symbols`, `occurrences`, `relations`, and `facts`
- `sourceMaps[*].mappings[*].sourceSpan`, `semanticSymbolId`, and `nativeAstNodeId`
- `sourcePreservation.tokens`, `trivia`, `directives`, `comments`, and exact source hashes
- `adapter.coverage` flags for exact AST, tokens, trivia, diagnostics, source ranges, generated ranges, and semantic coverage
- `losses[*].kind` for macro expansion, generated code, conditional compilation, dynamic runtime, unsupported syntax, source-map approximation, and source-preservation gaps
- `evidence[*].metadata` for parser version, compiler flags, project root, module/import roots, source hashes, payload hashes, and diagnostic counts

## Recommended Dependency-Free Improvements

1. Normalize external semantic-index format aliases more aggressively. Hosts already produce SCIP/LSIF/LSP/SemanticDB/Glean evidence under names such as `sourcegraph.scip`, `code-intelligence-protocol`, `language-server-index-format`, `lsp-document-symbols`, `semantic-db`, and `glean-schema`. Accepting these aliases keeps importer code dependency-free while avoiding evidence-only fallback.
2. Add a generic descriptor/IDL envelope for Protobuf, GraphQL, HCL, CSS, HTML, and WAT. These can import symbols, ranges, diagnostics, and source-preservation records without owning a runtime compiler.
3. Extend `NativeImportLanguageProfiles` only after an adapter evidence contract exists. For the harvested languages, start with blocked/needs-review profile rows rather than declaration regex scanners.
4. Add a `parserAstFormatIdForParser` alias pass for the harvested parser names (`solc`, `hclsyntax`, `protobuf-descriptor-set`, `graphql-js`, `libadalang`, `flang`, `dmd`, `lean4`, `racket-syntax`, `clojure-tools-analyzer`, `wasm-text`, `html-parser`, `css-syntax`) so host adapters appear in the parser AST matrix before native implementation.
5. Treat macro systems and generated outputs consistently: OCaml PPX, Julia/Nim/D/Lean/Racket/Clojure macros, Solidity/Yul generation, and Protobuf plugin output should all attach feature-evidence keys before becoming merge-ready.

## Local Outcome

Accepted a small, dependency-free alias-normalization improvement for external semantic indexes. It keeps source evidence in the SCIP/LSIF/LSP/SemanticDB/Glean import paths instead of falling through to the generic evidence-only payload importer when hosts use common long-form format names.
